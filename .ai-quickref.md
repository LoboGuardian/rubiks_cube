# AI Assistant Quick Reference

## 🚨 Critical Rules

1. **ALWAYS use `poetry run`** - Never run Python directly
2. **NEVER import OpenGL/pygame in `cube_model.py`** - Model must be GUI-independent
3. **ALWAYS draw ALL 6 faces** of each cube piece - Cube must be solid, not transparent
4. **Stickers go ON TOP**, not inside - Raise stickers above black plastic base

## 📁 File Responsibilities

| File | Purpose | Allowed Imports |
|------|---------|-----------------|
| `cube_model.py` | Business logic | `numpy`, `random`, `typing` ONLY |
| `cube_renderer.py` | OpenGL rendering | `OpenGL`, `pygame`, `numpy`, `cube_model` |
| `rubiks_cube.py` | Event handling | `pygame`, `cube_model`, `cube_renderer` |

## 🎨 Visual Requirements

The cube MUST look like this:

```
┌─────────────────────┐
│  Black plastic base │  ← Each cube piece
│  ┌───────────────┐  │
│  │ Colored       │  │  ← Sticker (raised 0.015 units)
│  │ Sticker       │  │  ← Inset to 88% of face
│  └───────────────┘  │
└─────────────────────┘
     Black border       ← Visible gap (CUBE_GAP = 0.05)
```

**Checklist**:
- ✅ Solid pieces (draw all 6 faces)
- ✅ Stickers raised above surface
- ✅ Stickers inset (88% of face size)
- ✅ Black borders visible
- ✅ Small gaps (0.05) between pieces

## 🏗️ MVC Architecture

```
User Input
    ↓
Controller (rubiks_cube.py)
    ↓
Model (cube_model.py) ←→ View (cube_renderer.py)
    ↓                        ↓
  Logic                   Display
```

**Data Flow**:
1. User presses key → Controller
2. Controller calls Model.rotate_face()
3. Controller gets Model.get_all_pieces()
4. Controller passes pieces to View.render()
5. View draws pieces with OpenGL

## 🔧 Quick Fixes

### Transparent Cube?
```python
# In cube_renderer.py → _draw_cube_piece()
# Make sure you're drawing ALL faces, not just colored ones:
for indices, color_idx in faces:
    face_vertices = vertices[indices]
    color = piece.colors[color_idx]
    is_sticker = (color != COLORS['BLACK'])
    self._draw_cube_face(face_vertices, color, is_sticker)  # Draw EVERY face
```

### Stickers on Wrong Side?
```python
# In cube_renderer.py → _draw_cube_face()
# Normal should point OUTWARD:
normal = np.cross(v1, v2)
normal = normal / np.linalg.norm(normal)
sticker_vertices = inset_vertices + normal * 0.015  # POSITIVE offset (outward)
```

## 🚫 Common Mistakes

❌ **DON'T**:
```python
# In cube_model.py
import pygame  # NEVER!
from OpenGL import *  # NEVER!

# In cube_renderer.py
if color != COLORS['BLACK']:
    self._draw_cube_face(...)  # DON'T skip black faces!
```

✅ **DO**:
```python
# In cube_model.py
import numpy as np  # OK
from typing import List  # OK

# In cube_renderer.py
for all faces:
    self._draw_cube_face(...)  # Draw EVERYTHING!
```
